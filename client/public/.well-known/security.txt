# Security Policy

Thank you for helping keep AgentX and our users safe!

## Reporting Security Issues

If you believe you have found a security vulnerability in any AgentX-owned repository, 
please report it to us through coordinated disclosure.

**Please do not report security vulnerabilities through public GitHub issues, discussions, or pull requests.**

Instead, please send an email to security@agentx.ai with the following information:

- Type of issue (e.g. SQL injection, XSS, etc.)
- Full paths of source file(s) related to the manifestation of the issue
- The location of the affected source code (tag/branch/commit or direct URL)
- Any special configuration required to reproduce the issue
- Step-by-step instructions to reproduce the issue
- Proof-of-concept or exploit code (if possible)
- Impact of the issue, including how an attacker might exploit it

## Safe Harbor

We support safe harbor for security researchers who:

- Make a good faith effort to avoid privacy violations, destruction of data, and interruption or degradation of our services
- Only interact with accounts you own or with explicit permission of the account holder
- Do not exploit a security issue for purposes other than verification

We will not pursue legal action against security researchers who follow these guidelines.

## Response Timeline

We will acknowledge receipt of your vulnerability report within 3 business days and will send a more detailed response within 7 business days indicating the next steps in handling your report.

We will keep you informed of the progress towards a fix and may ask for additional information or guidance.
